Você Sabia?
  • O Firebase SDK se comunica com o Firestore diretamente via gRPC, uma tecnologia de comunicação remota do Google — mais eficiente que APIs REST tradicionais.
  • O Firestore oferece Realtime Listeners — dados que se atualizam em tempo real no app quando o banco é modificado.
  • O FlutterFlow permite integrar APIs externas usando o recurso de Custom Actions e API Calls.

10.1 Como o FlutterFlow Consome o Firebase

Quando você configura o Firebase no FlutterFlow, a ferramenta gera automaticamente os métodos de acesso ao banco via a SDK do Firebase. Cada query configurada no Backend Query do FlutterFlow se traduz em uma chamada direta ao Firestore.

Definição

API (Application Programming Interface): conjunto de regras e protocolos que permite que sistemas diferentes se comuniquem. No caso do Firebase, a comunicação entre o app e o Firestore é feita via a SDK do Firebase, que abstrai as chamadas de rede e oferece funcionalidades como cache offline e sincronização automática.

10.2 Realtime Listeners: Dados em Tempo Real

O Firestore oferece Realtime Listeners, que permitem que o app receba atualizações automáticas quando os dados no banco mudam, sem precisar atualizar a tela manualmente.

No FlutterFlow, basta ativar a opção Real Time no Backend Query da tela de listagem. Quando um novo dado for inserido no banco (por outro usuário ou por outro dispositivo), a lista se atualiza automaticamente.

Caso de Uso do Realtime

O Realtime é ideal para apps colaborativos: listas de tarefas compartilhadas, chats, placar de jogos, monitoramento em tempo real. Se o seu app não tem necessidade de atualização em tempo real, você pode deixar o Realtime desativado para economizar recursos.

10.3 Integrações com APIs Externas

O FlutterFlow permite integrar qualquer API REST externa via API Calls. Isso abre possibilidades de enriquecer seu app com dados e funcionalidades de serviços de terceiros.

Exemplos de APIs Gratuitas para Projetos

Tabela 10.2 - APIs públicas e gratuitas para integração.
API O que oferece Exemplo de uso
ViaCEP Dados de endereço a partir do CEP Preenchimento automático de endereço
OpenWeatherMap Previsão do tempo App de atividades ao ar livre
Open Food Facts Informações nutricionais de alimentos App de dieta/nutrição
ExchangeRate API Taxas de câmbio em tempo real App de controle financeiro com câmbio
Google Maps API Mapas, rotas, localização App de entrega, localização de serviços

Configurando uma API Call no FlutterFlow

  1. Acesse API Calls no menu lateral.
  2. Clique em + Add API Call.
  3. Defina o nome, método HTTP (GET, POST) e a URL da API.
  4. Configure os parâmetros, cabeçalhos (headers) e o corpo (body) se necessário.
  5. Teste a chamada com o botão Test API Call.
  6. Mapeie os campos da resposta JSON para variáveis do FlutterFlow.
