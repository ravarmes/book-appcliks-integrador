11.2 Notificações Push com FlutterFlow

O FlutterFlow possui integração nativa com o Firebase Cloud Messaging (FCM) para envio de notificações push. As notificações podem ser enviadas para um usuário específico, para um grupo ou para todos os usuários do app.

Definição

Push Notification: mensagem enviada pelo servidor para o dispositivo do usuário, mesmo quando o app não está em uso. No Android, as notificações aparecem na barra de notificações e na tela de bloqueio.

A Figura 11.1 ilustra os três tipos de notificações disponíveis em apps mobile e seus casos de uso típicos.

Configurando Notificações no FlutterFlow

  1. Acesse Settings → Firebase no FlutterFlow e conecte o projeto Firebase.
  2. Ative o Cloud Messaging no console do Firebase.
  3. No FlutterFlow, use a Action Send Push Notification configurando: destinatário, título e corpo da notificação.
  4. Para notificações locais (sem servidor), use a Action Trigger Push Notification.

Tipos de Notificações

Tabela 11.2 - Tipos de notificações e seus casos de uso.
Tipo Quando Usar
Notificação Local Lembretes agendados (ex: lembrar de registrar transação diária)
Push (usuário específico) Alerta para o usuário (ex: limite de orçamento atingido)
Push (todos os usuários) Comunicados gerais (ex: nova versão disponível)

Como Funciona Tecnicamente o Push Notification

O envio de uma notificação push envolve uma cadeia de componentes que começa no dispositivo do usuário e termina no Firebase Cloud Messaging (FCM). O conceito mais importante a entender é o FCM Token — um identificador único gerado para cada instalação do app em um dispositivo específico. Pense nele como o "endereço de entrega" da notificação. Sem armazená-lo (geralmente em uma coleção no Firestore), não é possível enviar notificações para usuários específicos.
