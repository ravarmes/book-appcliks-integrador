SEMANA 12 – Tratamento de Erros e Preparação para Publicação

Nesta semana, você irá implementar o tratamento de erros no app e iniciar a preparação para publicação na Google Play Store, preenchendo os metadados necessários.

Um app pronto para publicação não é apenas um app que funciona — é um app que falha graciosamente. Erros acontecem: conexão cai, dados inválidos são inseridos, o servidor fica indisponível. Um app profissional trata esses erros de forma elegante, informando o usuário sem travar ou exibir mensagens técnicas.

Você Sabia?
  • Segundo a Google Play, apps com taxa de crash abaixo de 1% recebem prioridade no algoritmo de recomendação da loja.
  • A descrição do app na Play Store é um dos principais fatores de ASO (App Store Optimization) — o SEO dos apps.
  • Screenshots de alta qualidade aumentam a taxa de conversão (download) em até 35% na loja.

Fonte: Google Play Console; StoreMaven Research.

12.1 Tratamento de Erros no FlutterFlow

O FlutterFlow permite tratar erros em Actions usando os blocos de Error Handling. Toda Action que faz chamada ao banco de dados ou a uma API pode falhar — configure sempre o que acontece em caso de erro.

Padrões de Tratamento de Erros

Tabela 12.1 - Estratégias de tratamento de erros em apps mobile.
Tipo de Erro Estratégia Recomendada
Sem conexão com internet Exibir mensagem amigável e botão "Tentar novamente"
Token expirado / sessão inválida Redirecionar para tela de login automaticamente
Dado não encontrado Exibir estado vazio (empty state) com instrução clara
Erro de servidor (5xx) Exibir mensagem genérica "Tente novamente mais tarde"
Validação falhou Destacar campo com erro e exibir mensagem específica (semana 5)
Definição: Empty State

Empty State é a tela exibida quando não há dados para mostrar em uma lista ou seção. Um bom empty state indica ao usuário o que fazer para criar o primeiro registro. Exemplo: uma ilustração com o texto "Nenhuma transação ainda. Toque em + para adicionar."

Os 4 Estados de Tela — Design de UX de Qualidade

Uma tela profissional não tem apenas o estado "funcionando com dados". Ela trata explicitamente todos os estados que o usuário pode encontrar. A Figura 12.2 ilustra os 4 estados obrigatórios que devem ser implementados em toda tela com dados dinâmicos no FlutterFlow:

  1. Loading — enquanto a Backend Query carrega (use CircularProgressIndicator).
  2. Sucesso (Data) — quando há dados para exibir (seu ListView normal).
  3. Vazio (Empty State) — quando a query retorna 0 resultados.
  4. Erro — quando a chamada falha (sem internet, token expirado, servidor indisponível).

No FlutterFlow, cada um desses estados é configurado na seção Conditional Visibility dos widgets, utilizando variáveis booleanas de estado da página (Page State).

A Figura 12.1 mostra o padrão de tratamento de erros no FlutterFlow: cada Action pode ter dois desfechos — sucesso ou erro — e cada tipo de erro exige uma resposta diferente para o usuário.

Configurando Error Handling no FlutterFlow

  1. No painel de Actions, clique na Action que pode falhar (ex: Insert Row).
  2. Ative a opção Error Handling.
  3. No bloco de erro, adicione uma Action Show Snack Bar ou Alert Dialog com a mensagem de erro amigável.
  4. Evite exibir mensagens técnicas (como stack traces) para o usuário final.

12.2 Metadados para a Google Play Store

Para publicar o app na Play Store, você precisa preparar um conjunto de metadados (informações descritivas) que serão exibidos na página do app na loja.

Tabela 12.2 - Metadados obrigatórios para publicação na Play Store.
Campo Limite / Formato Dicas
Nome do app Máx. 30 caracteres Inclua palavras-chave relevantes
Descrição breve Máx. 80 caracteres Descreva o principal benefício do app
Descrição completa Máx. 4.000 caracteres Liste funcionalidades; use formatação simples
Screenshots 2 a 8 imagens (PNG/JPG, mín. 320px) Mostre as telas mais atrativas do app
Ícone PNG 512×512px Fundo sólido, sem transparência
Categoria Selecionar da lista Escolha a categoria mais próxima do tema
Classificação etária Questionário IARC Responder honestamente o questionário
Política de privacidade URL válida Obrigatório; pode ser Google Sites ou Notion
Política de Privacidade

A política de privacidade é obrigatória para qualquer app que colete dados de usuários (nome, e-mail, dados pessoais). Ela deve estar hospedada em uma URL pública e descrever quais dados são coletados, como são usados e com quem são compartilhados. Use o gerador gratuito em privacypolicygenerator.info como ponto de partida.

12.3 Screenshots e Gráficos Promocionais

As screenshots são a principal vitrine do app na Play Store e influenciam diretamente a decisão de download. Capture screenshots das telas mais importantes:

  • Tela inicial / dashboard
  • Tela de listagem principal
  • Tela de criação/formulário
  • Tela de detalhe
  • Tela de configurações
Dica: Screenshots com Moldura

Adicione as screenshots dentro de uma moldura de celular para um visual mais profissional. Use ferramentas como DaVinci, Previewed.app ou Figma para criar imagens promocionais com a moldura do dispositivo e texto explicativo.

12.4 Entregável da Semana

Resumo da Semana
  • Error handling implementado nas principais Actions do app (pelo menos nas operações de banco).
  • Metadados da Play Store preenchidos em documento (nome, descrição breve, descrição completa, categoria).

  • Política de privacidade criada e hospedada em URL pública.

12.5 Estudo de Caso: FinanCampus

Veja como o grupo do FinanCampus preparou o app para publicação.

📱 Estudo de Caso — Semana 12

Error handling implementado em 5 Actions:

  • Insert de transação: SnackBar "Não foi possível salvar. Verifique sua conexão."
  • Login: Alert Dialog "E-mail ou senha incorretos." (sem expor o erro técnico do Supabase)
  • Cadastro: Alert Dialog "E-mail já cadastrado."
  • Carregamento da lista: empty state com ícone e texto "Nenhuma transação ainda. Toque em + para começar!"
  • Perda de conexão: SnackBar "Sem conexão com a internet. Tente novamente."

Metadados da Play Store preenchidos:

  • Nome: FinanCampus — Controle de Gastos
  • Descrição breve: "Controle seus gastos mensais de forma simples e visual."
  • Categoria: Finanças

Política de privacidade: publicada em página do Notion (notion.so/financampus/privacidade), descrevendo coleta de e-mail e dados financeiros informados pelo usuário.
